# wk-im-dev 安装/初始化用户体验优化 — 计划

## 背景与目标

`wk-im-dev` 当前安装初始化流程对用户不够友好：

- 安装与初始化分两步，用户读完 install 输出后还要再跑 `wk-im-init.sh`
- `wk-im-init.sh` 需要显式 `--root`，没有合理默认
- 入口命令分散：`wk-im-dev`、`claude --agent wk-im-dev`、`/wk-im-dev:setup`、`wk-im-init.sh` —— 用户记不住
- 没有 `doctor` 排查命令
- CodeGraph 在 init 中间弹交互式 `[Y/n]` 阻塞流程
- shell rc 改完必须重开终端，新手踩坑
- README 491 行，新手 Quick Start 不"快"
- `install.sh` 五个 `--skip-*` flag 把 README"选项速查"挤满

### 验收标准

1. `curl … bootstrap.sh | bash -s -- --target $(pwd)` 跑完即可直接干活（自动 init + 知识库 ready）
2. `wk-im-init.sh` 无参数时能从 pwd 向上找 `.podspec`/`Podfile` 自动定位
3. `wk-im-dev` launcher 能自动选 Claude Code / Codex 运行时
4. `wk-im-dev doctor` 一行输出关键安装状态
5. CodeGraph 不再阻塞 init，默认改为提示性输出
6. 安装末尾打印一行可复制的 `export PATH=...` 命令，避免新会话踩坑
7. Claude Code agent 首次激活时检测 workspace.json 缺失会主动提示
8. README Quick Start 收敛到 ≤ 3 条命令，高级 flag 移到 `docs/advanced-install.md`

## 影响范围

| 模块 | 文件 | 变更类型 |
|---|---|---|
| 脚本 | `wk-im-dev/scripts/install.sh` | 新增 `--init` 默认开启逻辑 |
| 脚本 | `wk-im-dev/scripts/bootstrap.sh` | 透传 init 选项 |
| 脚本 | `wk-im-dev/bin/wk-im-init.sh` | 增加自动 root 探测；CodeGraph 默认禁用，加 `--with-codegraph` |
| 脚本 | `wk-im-dev/bin/wk-im-dev` | 改造为 multi-runtime dispatcher + doctor 子命令 |
| Agent | `wk-im-dev/agents/wk-im-dev.md` | 首次激活自检 workspace.json |
| 文档 | `wk-im-dev/README.md` | Quick Start 精简，高级 flag 抽走 |
| 文档 | `wk-im-dev/docs/advanced-install.md` | 新建，收纳 skip 类 flag |

## 实施步骤（分阶段）

### 阶段 1：脚本核心改造（落地 #1 #2 #5）
- 1.1 `install.sh`：加 `--init`（默认 on）/`--skip-init`，安装末尾若 target 看起来是 IM 仓库则自动调用 `wk-im-init.sh --root $TARGET --quiet`
- 1.2 `bootstrap.sh`：透传 `--skip-init` / `--with-codegraph`
- 1.3 `wk-im-init.sh`：
  - 无 `--root` 时从 pwd 向上找最近的 `.podspec` 或 `Podfile`（同时存在 BTIMService/BTIMModule 引用的 Podfile）
  - 找不到时读 `~/.wk-im-dev/workspace.json` 拿历史 service/module/host-app 作候选
  - CodeGraph 默认不安装、不 init；加 `--with-codegraph` 显式启用
- 1.4 `install.sh` 末尾改打印逻辑（落地 #8）：先打印可复制 `export PATH=...`，再打 launcher 绝对路径

### 阶段 2：launcher 多 runtime 化 + doctor（落地 #3 #4）
- 2.1 `bin/wk-im-dev`：
  - 检测 `claude` CLI + `~/.claude/settings.json` 含 wk-im-dev plugin → 转发 `claude --agent wk-im-dev "$@"`
  - 否则走原 Codex 分支
  - 加环境变量 `WK_IM_DEV_RUNTIME=claude|codex` 强制覆盖
  - 加子命令 `doctor` / `--doctor`：列出 launcher / core spec / codex agent toml / codex profile / workspace.json / shell rc PATH / codegraph 状态
- 2.2 不破坏现有 `wk-im-dev [codex options] [prompt]` 直接传入 codex 的语义

### 阶段 3：Agent 首次自检（落地 #7）
- 3.1 在 `agents/wk-im-dev.md` 增加"首次激活自检"段落，让模型在首次回复前检查 `~/.wk-im-dev/workspace.json` 是否存在，缺失则提示用户 `/wk-im-dev:setup`

### 阶段 4：文档收敛（落地 #6 #9）
- 4.1 新建 `wk-im-dev/docs/advanced-install.md`，收纳 `--skip-*` / `--replace-*` / `--no-shell-rc` 全部 flag 详解、AGENTS marker、profile marker、卸载细节
- 4.2 `wk-im-dev/README.md`：
  - 顶部 Quick Start 收敛到 3 行（Codex curl / Claude plugin install / 之后 `wk-im-dev`）
  - 删除/折叠重复的"安装详情"长段
  - "选项速查"只保留高频项，详表链到 advanced-install.md
- 4.3 加一行 `wk-im-dev doctor` 介绍

### 阶段 5：验证 + 同步 verify.sh（保障所有改动可回归）
- 5.1 跑 `bash scripts/verify.sh`
- 5.2 dry-run install 到一个临时目录验证 `--init` / `--skip-init` 路径
- 5.3 在干净仓库下手动跑 `wk-im-init.sh`（无参数）验证自动定位逻辑
- 5.4 `wk-im-dev doctor` 在已装 / 未装两种状态下都能输出

## 风险与回滚

| 风险 | 缓解 |
|---|---|
| `install.sh` 默认 `--init` 在用户跑 verify 用途的临时目录可能反而启动知识库扫描 | 增加 IM 仓库探测：仅当 target 含 `.podspec`(BTIMService/BTIMModule) 或 含 BTIM* 引用的 Podfile 才自动 init；否则只 install 不 init |
| launcher 多 runtime 探测错判（用户机器有 claude 但未装 plugin） | 严格要求 `claude plugin list` 或 settings.json 中存在 `wk-im-dev` 才走 claude 分支，否则回退 Codex；环境变量可强制 |
| README 精简后老用户找不到 flag | 所有移动到 advanced-install.md 的 flag 在 README 选项速查保留链接 |
| 自动向上找 podspec 误命中无关 Pod 仓库 | 仅匹配 `BTIMService.podspec` / `BTIMModule.podspec`；其他名字时不自动定位，回退到提示 |

回滚：所有改动均按阶段独立提交，单阶段可独立 revert。

## 验证方式

```bash
# 静态验证
cd wk-im-dev && bash scripts/verify.sh

# bootstrap 干跑（不安装到真实环境，仅看安装脚本能合并 init）
bash wk-im-dev/scripts/install.sh --target /tmp/fake-im-repo --runtime codex --skip-init  # 验证 skip
bash wk-im-dev/scripts/install.sh --target /tmp/fake-im-repo --runtime codex              # 验证默认（target 非 IM 仓 → 安装但不 init）

# launcher 自检
~/.wk-im-dev/bin/wk-im-dev doctor

# init 默认行为
cd /path/to/BTIMService && ~/.wk-im-dev/bin/wk-im-init.sh  # 应自动找到 root
```

- `feat(wk-im-dev): agent 首次激活自检 workspace.json`
