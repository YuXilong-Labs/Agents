# 2026-06-16 视频编辑 Agent 框架 计划

## 背景与目标

仓库已有「单一模板 + 生成器」框架(`tools/create-wk-agent.sh` + `manifests/*.json`,`wk-im-dev` 即模板)。
本次要新增一个 iOS **视频编辑** per-component agent `wk-video-dev`(双组件 Core + UI),
并顺手补齐生成器在「新增 Agent」全流程上的两个真实缺口。用户选定:全套 + 双组件 + 先改进生成器。

### 验收标准

- [ ] `tools/create-wk-agent.sh` 生成 agent 时能自动注册到根 `.claude-plugin/marketplace.json`(可 `--no-register` 关闭)。
- [ ] dogfood 不破:`--manifest manifests/im.json --out /tmp/regen` 与 `wk-im-dev/` 仍仅 4 个文件差异(两个 plugin.json / CHANGELOG / components.conf 注释头)。
- [ ] `manifests/video-edit.json` 存在且字段完整。
- [ ] 生成的 `wk-video-dev/` 通过自身 `scripts/verify.sh`。
- [ ] `wk-video-dev/` 已注册进 `marketplace.json`,且领域散文(README/architecture/knowledge topic/约束/示例)是视频域,非 IM 残留。
- [ ] 根 `README.md` 更新:版本纠正、补「如何新增 Agent」指南、列出 video agent。
- [ ] 仓库根垃圾文件(`AGENTS.md`、`CLAUDE-FABLE-5.md`)清理或 gitignore。

## 实施步骤(分阶段)

### 阶段 1 — 改进生成器(name 无关,先做)
1. `create-wk-agent.sh`:python 块增加 `REPO_ROOT` + `REGISTER` 入参;生成后若 `out` 的父目录 == 仓库根,则 upsert 进 `<repo>/.claude-plugin/marketplace.json`(name/description/keywords/source.path=basename)。加 `--no-register` flag。
2. 残留扫描:把模板 IM 隐私名词(messageBody/msgContent/attachmentURL)纳入「请人工 review」提示(不改文件,仅提示)。
3. (Tier B,确认后)把 `agents/wk-im-dev.md`、`codex/AGENTS.md`、`constraints*.md` 的隐私约束句改为泛化措辞(「隐私字段见 components.conf 的 privacy 项」),去掉硬编码 IM 名词。
4. 验证:dogfood 重生成仍 4 文件差异;`example-pay.json` 生成 + verify 通过 + 已注册进(临时副本的)marketplace。

### 阶段 2 — 写 video manifest
5. `manifests/video-edit.json`:slug=`wk-video-dev`,components=[VideoEditCore(service), VideoEditUI(module)],forbid_import=[Core→UI, UI→VideoEngineSDK],privacy=[sourceURL/exportPath/licenseKey/token/accessToken],sdk_literal=VideoEngineSDK。**名字/规则待用户确认。**

### 阶段 3 — 生成 + 注册
6. `tools/create-wk-agent.sh --manifest manifests/video-edit.json --out wk-video-dev`(自动注册 marketplace)。
7. 跑 `wk-video-dev/scripts/verify.sh` 应过。

### 阶段 4 — 视频领域散文重写
8. 2 个 skill 目录 `im-knowledge`→`video-knowledge`、`im-review`→`video-review`,并改 `agents/wk-video-dev.md` 路由表引用。
9. 重写:`README.md` 提需求示例/工作流/约束表、`docs/architecture.md`、`skills/video-knowledge/*`(message-flow→视频导出/编辑 pipeline)、`agents/wk-video-dev.md` 的 Identity/跨组件示例、`codex/AGENTS.md` 同步。
10. 再跑 verify;`bash scripts/install.sh --runtime both`(临时 target)+ `~/.wk-video-dev/bin/wk-video-dev doctor` 冒烟。

### 阶段 5 — 文档 + 清理
11. 根 `README.md`:纠正 wk-im-dev 版本、补「新增 Agent = 写 manifest → generate(自动注册)→ 重写散文 → verify → tag」、列 wk-video-dev。
12. `tools/README.md` 补「生成后接入仓库」小节。
13. 清理根 `AGENTS.md`、`CLAUDE-FABLE-5.md`(删除并按需加 `.gitignore`)。
14. 更新项目 `CLAUDE.md` 模块表(加 wk-video-dev 行)。

## 风险与回滚

- **pod 名猜错** → 阶段 4 散文白写。缓解:阶段 2/3 前用户确认名字;万一改名,改 manifest 一行重跑生成(阶段 3 幂等),散文按 diff 局部修。
- **改模板(Tier B)影响 wk-im-dev 行为** → 仅措辞泛化,guard 实际仍读 components.conf(正确);dogfood 验证兜底。回滚 = git revert 那几个文件。
- **marketplace JSON 写坏** → 生成器用 python json 读写 + verify 的 check_json 兜底;回滚 = git checkout marketplace.json。
- **wk-video-dev 暂不发 tag**:本次只到「本地 verify + doctor 冒烟」,端到端 tag 发布(CLAUDE.md 步骤 3)等用户真实仓库联调后再做。
