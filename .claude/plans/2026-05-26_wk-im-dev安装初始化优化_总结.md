## 风险与遗留

- **未在真实 IM 仓库做端到端验证**：当前 dry-run 都在 fake `BTIMService.podspec` 上，真实仓库可能触发 codegraph init 等更重的路径。建议在 BTIMService / BTIMModule / HostApp 各跑一次完整 curl 安装。
- **`wk-im-kb-check.sh` 首次扫描产生"未链接 topics"警告**：根因是 kb-scan 创建了 topics 文件但未在 index.md 添加链接。这是 kb 层既有问题，不在本计划范围；当前缓解是 install 不再当 fatal 处理。如果想根治，需要让 kb-scan 在 generated block 里自动添加 topics 链接。
- **CodeGraph 已安装时仍会自动 init 缺失索引**：保留这个便利但用户若不想要可手动删 `.codegraph/`。
- **Claude Code 路径下 `wk-im-dev` launcher 需要 PATH 生效**：新会话需 source shell rc 或用 install 末尾打印的 export 命令。doctor 已显式提示。

## 后续可选优化
