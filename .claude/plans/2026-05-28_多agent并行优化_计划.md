# 2026-05-28 wk-im-dev 多 agent 并行优化 计划

## 背景与目标

经过对 wk-im-dev 7 个 subagent + 6 个 skill 的全面 review，发现主 agent 已经有较完整的多 agent 架构（explorer / planner / debugger / executor / verifier / knowledge-maintainer 职责清晰、读写隔离），但有 3 处并行机会没吃满。

### 验收标准

| 优化点 | 期望 |
|---|---|
| P1 verifier 并行 | Build/Test 与 Guard/Knowledge/Diff Scope 等独立维度并行启动；典型场景验证总耗时砍半 |
| P2 explorer 单组件并行 | 单组件内任务若涉及 ≥3 个独立子系统/topic，主 agent 能并行派出多个 explorer |
| P3 debugger 多假说并行 | crash 有 ≥2 个相互独立的可疑根因时，并行派出多个 debugger 各验一个假说 |
| 身份介绍优化 | 主 agent 问候模板换成能力 + 示例版（已落地） |
| 双 runtime 一致 | 改动同步到 `core/wk-im-dev-core.md`，确保 Claude/Codex 行为对齐 |

## 影响范围

| 文件 | 改动类型 | 内容 |
|---|---|---|
| `agents/wk-im-dev.md` | prompt 改动 | 问候模板（已改） + 路由规则增加 explorer 单组件并行启发式 |
| `agents/wk-im-explorer.md` | prompt 改动 | description 增加"单组件多子系统并行"语义 |
| `agents/wk-im-debugger.md` | prompt 改动 | 增加"多假说并行验证"段落 |
| `agents/wk-im-verifier.md` | prompt 改动 | 增加"并行执行规则"段，列出独立维度集合，要求主 agent 同时启动 |
| `core/wk-im-dev-core.md` | 同步改动 | Identity / Subagent Roles 段同步上述变化 |

**不改动**：
- skill 文件（feature/bugfix/im-review 等）——它们已经在文本里说了"委派 explorer/planner/...subagent"，subagent 自身行为改了之后 skill 自动获益
- bin/scripts/hooks——无脚本改动
- 不增加新的 agent 文件，不破坏现有 7 个 agent 的拓扑

## 风险与回滚方案

| 风险 | 缓解 |
|---|---|
| verifier 并行 Bash 的依赖关系处理不当 | 只把"已知独立"的维度并行（Build/Test、Guard、Knowledge、Diff Scope）；有依赖的（Impact 依赖 codegraph_impact 输入）仍串行 |
| explorer 单组件多并行可能产生重叠结果 | 主 agent 在 prompt 中负责合并去重，并在路由规则里明确"任务关键词能列出 ≥3 个独立子系统"才触发 |
| debugger 多假说并行可能产生互斥结论 | 主 agent 收敛阶段择"证据最强"项，并把其他假说作为已排除项写入根因报告 |
| 双 runtime 不一致 | core 文件同步改动；install.sh 同时装到 ~/.claude 和 ~/.codex |
| 改动太集中触发回归 | 改动均为 prompt 文本，无脚本/契约变化；verify.sh + 实际 install 后人工跑一次 setup 自检 |

**回滚**：所有改动都是文本编辑，`git revert` 即可。
