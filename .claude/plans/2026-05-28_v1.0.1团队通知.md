# 📢 wk-im-dev v1.0.1 正式版发布通知

各位 iOS 同事，

`wk-im-dev` 这个专门做 BTIMService / BTIMModule 的 AI 开发 agent 完成 **v1.0.1 首个正式版本**，欢迎大家直接装来用。

## 这是什么？

一个集成在 **Codex CLI** 和 **Claude Code** 里的 iOS IM 开发助手，能：
- 帮你**开发新功能**（消息 / 会话 / 聊天 UI 等）
- **定位 crash、性能、状态机异常**根因
- **审查代码改动**、PR diff
- 解答**架构、消息流程、跨 Pod API 契约**等问题

内部由 7 个子 agent（explorer / planner / debugger / executor / verifier / knowledge-maintainer）协作完成任务，对外只需要你描述目标即可。

## v1.0.1 亮点

- ⚡ **多 agent 并行能力补完**：explorer 可同时探索多个独立子系统、debugger 可并行验证多个根因假说、verifier 把独立验证维度同时跑，**验证总耗时砍半**
- 🎯 **主 agent 身份介绍重写**：会告诉你它能做什么、给你几个示例任务
- ⚙️ 没有配置 workspace 时会自动提醒你跑 `/wk-im-dev:setup`

完整发布说明：<https://github.com/YuXilong-Labs/Agents/blob/main/wk-im-dev/CHANGELOG.md>

## 快速安装

### 我用 Codex（最常见）

在 BTIMService 或 BTIMModule 仓库根目录执行：

```bash
curl -fsSL https://raw.githubusercontent.com/YuXilong-Labs/Agents/v1.0.1/wk-im-dev/scripts/bootstrap.sh \
  | bash -s -- --target . --ref v1.0.1
```

### 我用 Claude Code

```bash
claude plugin marketplace add YuXilong-Labs/Agents
claude plugin install wk-im-dev@yuxilong-agents
```

### 我两个都用

```bash
curl -fsSL https://raw.githubusercontent.com/YuXilong-Labs/Agents/v1.0.1/wk-im-dev/scripts/bootstrap.sh \
  | bash -s -- --target . --ref v1.0.1 --runtime both
```

## 装完三步自检

```bash
# 1. 版本号应输出 1.0.0
wk-im-dev --version

# 2. 全套环境自检（应全 [ok]）
wk-im-dev doctor

# 3. 启动会话，问个问候确认 agent 活着
wk-im-dev
> 你好，你是谁？
```

正常情况下你会看到：

> 你好，我是 wk-im-dev——BTIMService 与 BTIMModule 的专属开发 agent。
>
> 可以帮你：
> - 开发新功能（消息 / 会话 / UI）
> - 定位 crash、性能、状态异常
> - 审查代码改动、PR diff
> - 解答架构、消息流程、API 契约
>
> ...

如果你**还没初始化 workspace**（首次装一定是这样），它会追加一行：

> ⚠️ 还没检测到 workspace 配置，建议先 `/wk-im-dev:setup` 初始化。

照做即可，初始化一次后就一直生效。

## 第一次用，怎么提需求？

直接说，agent 会自动选择对应的子流程。例如：

| 想做什么 | 怎么说 |
|---|---|
| 开发新功能 | "加一个消息撤回功能"、"支持发送语音转文字" |
| 修 bug | "未读数不准"、"切换会话偶发 crash，堆栈在 XXX" |
| 审查代码 | "看下这个 PR / 这次改动有没有问题" |
| 问架构 | "消息发送后 UI 是怎么刷新的？"、"BTIMService 和 BTIMModule 怎么交互" |

## 之前装过 pre-1.0 版本（v3.x.x）的同事

**v3.0.0 → v3.5.0 全是开发期内部版本号，已从 GitHub 删除对应 tag**。如果你之前装过，重装 v1.0.1 即可：

```bash
# 清掉本地残留的旧 tag
git fetch --prune --prune-tags origin

# 重新装
curl -fsSL https://raw.githubusercontent.com/YuXilong-Labs/Agents/v1.0.1/wk-im-dev/scripts/bootstrap.sh \
  | bash -s -- --target . --ref v1.0.1
```

Claude Code 用户跑 `claude plugin update wk-im-dev@yuxilong-agents` 即可。

> 💡 **预防说明**：之前装过 pre-1.0 plugin 的 Claude Code 用户跑 update 时会看到类似 `Plugin "wk-im-dev" updated from 3.3.0 to 1.0.1` 的提示——这是**正常的版本号体系重置**，不是 downgrade。新装的同事不会看到这条。

## 出问题怎么办

1. **第一步**：跑 `wk-im-dev doctor`，按它的 `Suggested fixes` 提示做（90% 的问题它能自检 + 自动提示修复命令）
2. **第二步**：看排错速查 → <https://github.com/YuXilong-Labs/Agents/blob/main/wk-im-dev/docs/team-distribution.md#4-排错速查>
3. **第三步**：还搞不定就直接在群里 @我 / 提 issue 到 <https://github.com/YuXilong-Labs/Agents/issues>
