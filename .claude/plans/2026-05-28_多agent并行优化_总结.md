## 风险与遗留

- **效果依赖 LLM 的并行调度能力**：所有改动均为 prompt 文本，实际并行度取决于 model（Sonnet/Opus 表现可能不同）。后续观察实际行为。
- **未拆分 executor**：因违反 cross-component change ordering（service 必须先于 module）保持单 writer，未来若有同组件独立目录可考虑。
- **未拆 verifier 为多 agent**：选了"单 agent 内 Bash 并行"的轻方案，避免增加 agent 数量。如果未来观察到并发未充分利用，再考虑物理拆分为 verifier-build / verifier-static。
